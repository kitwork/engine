Parallel Goroutines

Unlike NodeJS which relies on a single-threaded Event Loop, Kitwork empowers you to spin up real, isolated multi-threaded processes directly from your JavaScript Code.

The `go()` Function

Whenever you wrap an execution block inside go(), Kitwork safely extracts that block and executes it across its Sovereign VM pool.

router.get("/register").handle((request, response) => { // 1. Save info immediately const data = request.body(); // 2. Spin up Async background worker go(() => { // This executes separately. // Response will NOT wait for this. network.sendEmail(data.email, "Welcome!"); log.Print("Welcome Email sent to: " + data.email); }); // 3. Return response instantly (0ms latency!) return response.json({ success: true }); });

Why use Goroutines?

Standard Javascript runtimes block the execution thread while hashing passwords, processing images, or computing heavy loops. The `go()` keyword eliminates these bottlenecks entirely. Never block your HTTP routers again.
